# Architecture UniThread

UniThread — микросервисная Q&A-платформа для университета.

## Домены

- `auth-service` — пользователи, роли, JWT.
- `ticket-service` (логически question-service) — вопросы и треды комментариев.
- `review-service` — отзывы и репутация пользователей.
- `frontend` — единый UI для USER и ADMIN.

## Поток авторизации

1. Пользователь логинится через `auth-service`.
2. Получает JWT с `userId`, `username`, `role`, `banned`.
3. Frontend отправляет JWT в `Authorization: Bearer ...`.
4. Остальные сервисы валидируют JWT локально по общему `JWT_SECRET`.

## Поток вопросов и обсуждений

1. USER создает вопрос.
2. Вопрос отображается в общей ленте.
3. Другие USER оставляют комментарии.
4. ADMIN модерирует вопросы и комментарии.

## Репутация

1. USER оценивает другого USER.
2. REVIEW-сервис хранит рейтинг и отзывы.
3. Профиль показывает средний рейтинг и список отзывов.

## CI/CD и Kubernetes

Проект сохраняет микросервисную структуру для CI/CD и деплоя в Kubernetes.
Даже если папка называется `ticket-service`, в предметной области это сервис вопросов (`question-service`).
